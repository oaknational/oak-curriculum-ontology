# Contributing to Oak Curriculum Ontology

Thank you for your interest in the Oak Curriculum Ontology!

## Current Status: Early Release (v0.1.0)

This is an **early public release** (v0.1.0) for evaluation and community feedback. The ontology structure, URIs, and data are under active development.

## How to Contribute

### ✅ We Welcome Feedback

We value your input! Please share:

- 🐛 **Bug reports** - Issues with data quality, structure, or documentation
- 💡 **Feature suggestions** - Ideas for improvements or additions
- 📝 **Documentation improvements** - Clarifications, corrections, or enhancements
- ❓ **Questions** - About the ontology, data model, or usage

**How to provide feedback:**

1. [Open an issue](https://github.com/oaknational/oak-curriculum-ontology/issues/new) on GitHub
2. Use a clear, descriptive title
3. Provide as much context as possible:
   - What you were trying to do
   - What you expected
   - What actually happened
   - Relevant file paths or URIs

### ⚠️ Pull Requests Not Accepted (Yet)

**We are not accepting pull requests during this early release phase.**

Why? During v0.1.0, we are:
- Actively refining the ontology structure
- Establishing governance processes
- Validating the data model
- Setting up contribution workflows

## What Feedback Is Most Helpful?

### Data Quality Issues

Examples:
- "The Mathematics KS3 content descriptor for 'Fractions' has incorrect URI pattern"
- "Programme X references a year group that doesn't exist"
- "SHACL validation fails on valid data in biology-key-stage-4.ttl"

### Ontology Structure

Examples:
- "The relationship between Unit and Lesson is unclear"
- "Missing property to link Programmes to Exam Boards"
- "Ambiguity in how Threads relate to Units"

### Documentation

Examples:
- "SPARQL example on line 235 of README doesn't work"
- "File structure diagram is outdated"
- "Standards compliance doc mentions wrong filename"

### Use Case Requirements

Examples:
- "We need a property to track prerequisite units"
- "Missing metadata for tracking last updated timestamps"

## What We Can't Help With

- **General RDF/SPARQL support** - Please use Stack Overflow or RDF community forums
- **Custom ontology development** - This is specific to Oak/National Curriculum
- **Infrastructure setup** - For Neo4j, triple stores, etc., consult their documentation

## Code of Conduct

We expect all contributors to:
- Be respectful and constructive
- Focus on the technical merits of suggestions
- Assume good faith in discussions
- Keep feedback relevant to the ontology

## Questions?

If you're unsure whether something is appropriate feedback, go ahead and [open an issue](https://github.com/oaknational/oak-curriculum-ontology/issues/new) - we're happy to discuss!
